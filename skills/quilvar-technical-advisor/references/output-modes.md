# Output Modes

Use these patterns when the user wants advice framed for a specific audience.

## Executive Brief

Use for: executive pitch, board update, leadership summary, launch status.

Structure:
- What it is.
- Why it matters.
- What is working.
- What is not ready.
- What we should do next.

Keep it business-first. Mention technical details only when they change risk, cost, timing, or credibility.

## Launch Readiness

Use for: "Can we launch this?", "Is this production ready?", "What blocks release?"

Structure:
- Verdict: prototype, pilot-ready, internal-only, or externally launchable.
- Top blockers.
- Risks if launched now.
- Minimum hardening needed.
- Recommended release posture.

Be decisive. Avoid hedging unless uncertainty is genuinely material.

## Decision Memo

Use for: prioritization, sequencing, tradeoffs, roadmap debates.

Structure:
- Recommendation.
- Why this is the right choice now.
- What alternatives were considered.
- Cost of delay.
- Risks and mitigation.

Prefer a single clear recommendation over a menu of equal-sounding options.

## Investor / Founder Explanation

Use for: "What is the pitch?", "Why is this strategically valuable?"

Emphasize:
- speed to signal
- structured intelligence from noisy data
- validation feedback loop
- compounding data asset
- advisory, not auto-trading

Do not imply proprietary edge unless the current implementation actually supports it.
